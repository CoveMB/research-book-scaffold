# Protocols

Use protocols to write down the rules for a research pass before the rules get
blurry.

This folder is for procedures that govern how sources will be found, selected,
extracted, coded, reviewed, or audited. A protocol can be short for a narrow task
or detailed for a systematic search, but make the method clear enough
that another pass could follow the same rules.

Include:

- Research question, scope, and boundaries.
- Search venues, planned query families, and citation chasing plan.
- Inclusion and exclusion criteria.
- Screening steps and decision rules.
- Extraction fields, coding rules, and quality checks.
- Review, audit, or stopping criteria.
- Known limits, assumptions, and follow-up tasks.

Use a protocol before starting a bounded search, extraction pass, citation audit,
source quality review, or literature mapping task. Update it when the method
changes, but record meaningful changes instead of silently rewriting history.

Do not use protocols as evidence. They describe how evidence will be handled; the
evidence itself belongs in source notes, extraction tables, matrices, claim notes,
or bibliography records.

# Source matrices

Use source matrices to compare sources against each other.

This folder is for cross-source judgment: what each source claims, how it knows
what it claims, where it is strong, where it is weak, and how it relates to the
project's questions. A matrix is useful once several sources have been selected
and the project needs a clearer view of agreement, conflict, gaps, and relevance.

Include:

- Citekey or source identifier.
- Source type, method, evidence type, and scope.
- Main claim or finding.
- Strengths, limits, and risks of misuse.
- Relevant concept notes, claim IDs, chapter briefs, or research questions.
- Notes on disagreement, missing evidence, or sources that need checking.

Use `templates/source-matrix-template.md` as the starting point when it fits. Keep
source-specific summaries and quotations in `notes/01-source-notes/` or
`research/extraction-tables/`; the matrix should make comparison easier, not
replace the underlying notes.

Do not turn a matrix into draft prose. If the comparison supports a claim, move
that claim into `notes/04-claim-ledger/` with its evidence status and remaining
risks.

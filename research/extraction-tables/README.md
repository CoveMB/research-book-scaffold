# Extraction tables

Use extraction tables to pull comparable evidence out of sources.

Use this folder when several sources need to be compared on the same fields, or
when a single source contains many passages, cases, variables, events, or claims
that need careful tracking. Keep general reading notes in
`notes/10-evidence/source-notes/`.

Include:

- Source identifier or citekey.
- Page, section, timestamp, table, figure, or other location reference.
- Verified quotation, paraphrase, summary, or coded value.
- Evidence type, method, population, case, period, or other fields relevant to the project.
- Notes on uncertainty, limits, exclusions, and decisions made during extraction.
- Related claim IDs, concept notes, chapter briefs, or follow-up tasks.

Extraction tables help keep evidence handling consistent. They are especially
useful before writing a source matrix, synthesis memo, claim note, or chapter
brief because they keep the raw extracted material separate from interpretation.

Do not use this folder to store unsupported conclusions. If a row depends on a
specific passage, include the location. If the source does not support the field,
say so instead of filling the cell from memory.

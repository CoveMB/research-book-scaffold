# Audit records

Use `audits/` for citation, claim, continuity, source-quality, placeholder,
release-readiness, rights, privacy, and final manuscript checks.

Create an audit note when a review has findings that should remain visible. Use
`templates/audit-template.md` when it fits.

Include:

- Audit scope and files checked.
- Checks run and checks skipped.
- Findings, severity, and required fixes.
- Evidence gaps or unresolved risks.
- Decision and follow-up tasks.

Separate blockers from warnings and environmental limits. Do not silently
rewrite around a failed audit. Record the issue, fix the underlying problem, and
rerun the relevant check.

# Audits

Use `90-audits/` for review records.

Audits sit outside the main writing flow on purpose. They preserve what was
checked, what failed, what was skipped, and what still needs attention.

Subfolders:

- `audits/` for citation, claim, continuity, source-quality, placeholder,
  release-readiness, rights, privacy, and final manuscript audits.

Create an audit when a problem should remain visible after the first pass:
missing support, unresolved citations, weak claims, continuity drift, external
reference failures, copied-text risk, or skipped checks.

Do not silently rewrite around a failed audit. Record the issue, fix the
underlying problem, and rerun the relevant check.

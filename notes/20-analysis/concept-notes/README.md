# Concept notes

Use `concept-notes/` for terms, concepts, distinctions, and boundaries that need
to stay stable across the project.

Create a concept note when a term is doing real work in source notes, claims,
briefs, or chapters. Use `templates/concept-note-template.md` when it fits.

Include:

- Working definition.
- Competing definitions or usages.
- Related sources and source notes.
- What the concept cannot support.
- Risks of ambiguity, overextension, or misuse.

A concept note is not a claim note. When a definition supports a manuscript
claim, the claim still needs source notes, citekeys, and locators.

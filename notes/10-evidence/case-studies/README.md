# Case studies

Use `case-studies/` for bounded cases, examples, episodes, organizations, sites,
or events that may later support a claim, chapter, or comparison.

Create a case note when the unit of analysis is the case itself rather than one
source. A case note can draw on several source notes, but it still needs a clear
source trail.

Include:

- The case boundary: what is inside the case and what is outside it.
- Chronology, relevant context, and source trail.
- Why the case matters for this project.
- Comparison logic, if the case will be compared with others.
- Interpretive limits and unresolved questions.

Do not generalize from a case without naming the basis for comparison. If missing
counter-cases would weaken the use, mark that gap.

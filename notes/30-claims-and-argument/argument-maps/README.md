# Argument maps

Use `argument-maps/` to test how claims connect before they become chapter order
or draft prose.

Create an argument map when several claims need sequencing, when a chapter's
logic feels fragile, or when objections need to be made explicit.

Include:

- Major claims and their order.
- Warrants, dependencies, and assumptions.
- Objections and counterarguments.
- Weak links.
- Evidence paths back to claim notes and source notes.

Do not use an argument map to upgrade unsupported claims. Keep uncertain links
marked until the evidence path is checked.

# Audits

Citation, claim, continuity, source-quality, and final checks.

Use this folder for review records that need to remain visible after problems are
found.

Include the audit scope, files checked, findings, severity, required fixes,
decision, skipped checks, and follow-up tasks. Separate likely blockers from
warnings or environmental limits.

Do not silently rewrite around a failed audit. Record the issue, then fix the
underlying citation, claim, placeholder, link, continuity, source-quality, or
release-risk problem.
